# 《催化剂》游戏开发规划文档

## 1. 游戏概述

**游戏名称**：《催化剂》（Catalyst）

**核心概念**：一款回合制策略规划游戏。玩家在一个充满惰性元素的微缩世界中，通过放置"转化规则"来编织连锁反应。游戏目标不是直接击败敌人，而是启动一个能够自我滋养、持续运转的连锁系统，看着反应积分雪球式增长，获得"连绵雨"式的满足感。

**核心乐趣**：
- "我算到了"——所有连锁反应都源于玩家的规划，而非直接操作
- "动态适配"——根据手中随机获得的规则牌，寻找最优切入点
- "连锁引爆"——早期放置的伏笔在后续产生远超预期的爆发
- "连绵雨"——连锁反应持续不断，积分从个位数累积到成千上万

**参考游戏**：
- 《杀戮尖塔》——构筑感、路线选择
- 《陷阵之志》——预判系统运行、完美解场
- 《神界：原罪2》——地表/元素交互系统
- PEAK——规划与分工协作感

## 2. 核心机制

### 2.1 世界构成

游戏主界面为一个网格地图（建议初始尺寸 8×8，可扩展）。

**基础元素类型**（惰性状态，等待被转化）：
- **水体**（Water）——蓝色，可流动
- **岩石**（Stone）——灰色，固定障碍
- **土壤**（Earth）——棕色，可种植
- **空气**（Air）——不可见，占据空格
- **植物**（Plant）——绿色，可生长
- **矿石**（Ore）——金色，稀有资源
- **冰晶**（Ice）——浅蓝，低温状态
- **熔岩**（Lava）——红色，高温状态（通常由转化产生）

**元素状态系统**：
- 每个格子可以同时拥有"基础元素"和"附加状态"
- 附加状态包括：燃烧、导电、冻结、光尘覆盖、蒸汽笼罩、带电、祝福等
- 状态之间具有交互规则（见下文）

### 2.2 规则卡牌系统

玩家手牌为"转化规则"，每回合获得一定数量的随机规则牌。玩家每回合可以往世界中放置1-3条规则（根据能量值）。

**规则卡牌类型**：

**A. 转化规则（Transform）**
格式：`当 [条件元素] 接触到 [触发元素/状态] 时，转化为 [结果元素]`
- 示例："当水体接触到熔岩时，转化为蒸汽"
- 示例："当岩石被光尘覆盖时，转化为矿石"

**B. 增殖规则（Multiply）**
格式：`当 [元素] 被 [触发条件] 时，向相邻格复制自身`
- 示例："当植物被蒸汽笼罩时，向所有相邻空格复制自身"
- 示例："当冰晶被带电时，向随机相邻格生成新冰晶"

**C. 触发规则（Trigger）**
格式：`当 [元素A] 生成时，对范围内 [元素B] 施加 [效果]`
- 示例："当蒸汽生成时，对2格范围内的所有植物施加'加速生长'（立即增殖一次）"
- 示例："当矿石生成时，对全图所有冰晶施加'光尘覆盖'"

**D. 灭绝规则（Extinction）**
格式：`当 [元素] 满足 [条件] 时，消失并产生 [效果]`
- 示例："当水体被燃烧状态覆盖时，消失并在原地生成蒸汽"
- 示例："当植物数量超过10时，多余的植物消失并转化为能量"

### 2.3 连锁反应触发机制

**核心循环**：放置规则 → 世界状态变化 → 触发更多规则 → 产生更多变化 → 连续反馈

**关键设定——"泛用物质"**：
- 引入一种名为"**催化剂尘**"的特殊状态，它不是具体元素，而是一种"元状态"
- 每当连锁反应发生一次，就向世界随机一格播撒"催化剂尘"
- 催化剂尘的效果：**降低该格元素触发规则的门槛**（比如让原本需要3个条件的规则只需1个条件就能触发）
- 这保证了随着连锁规模扩大，反应会越来越容易触发，形成正反馈雪球

**连锁积分系统**：
- 屏幕右上角显示【连锁反应次数】计数器
- 每次任意规则触发，计数器+1
- 计数器有视觉和音效反馈：10连/50连/100连/500连/1000连各有特殊提示
- 音效从零星的"叮"逐渐变成密集的"叮叮叮叮叮"，最终形成连续的嗡呜声
- 视觉上，屏幕边缘逐渐泛起光芒，反应区域出现越来越华丽的粒子特效

## 3. 游戏流程

### 3.1 回合结构

每个回合分为三个阶段：

**阶段1：观察**
- 玩家查看当前世界状态
- 查看手牌（5张规则牌，可保留至多3张到下回合）
- 查看预判提示（系统显示下一回合可能随机生成的基础元素）

**阶段2：布局**
- 玩家消耗能量从手牌中向世界网格放置规则
- 能量上限初始为3，可通过特定事件增加
- 规则被放置在具体格子上，成为"规则柱"，持续生效
- 规则柱有生命周期（通常3-5回合后失效），迫使玩家不断更新布局

**阶段3：演化**
- 玩家点击"执行"按钮
- 世界按照当前所有规则自动演化
- 连锁反应开始，积分计数器开始跳动
- 演化持续进行，直到达到稳定状态（30秒无新反应触发则停止）
- 玩家观看全程，感受"连绵雨"

### 3.2 胜利与失败条件

**胜利条件**（目标导向，类似音游的"通过一首歌"）：
- 每个关卡有"连锁目标分数"（如达到500连）
- 达成目标即可通关，解锁下一关卡
- 额外目标：在限定回合数内达成、使用特定规则组合达成、达到特定峰值连击数

**失败条件**（引入"动态适配"的张力）：
- 世界进入"死寂状态"：连续10回合无任何反应触发（规则用尽或组合失效）
- 世界进入"混沌失控"：连锁反应过快导致世界格子全部被单一元素占据（需要灭绝规则来制衡）
- 玩家手牌枯竭且无法维持现有规则柱

### 3.3 关卡设计

**渐进式关卡**（共4大区域，参考PEAK的生物群落概念）：

**区域1：海岸/启蒙**
- 地图6×6
- 仅有水体、岩石、土壤三种基础元素
- 目标：100连
- 教学：学习转化与增殖基础

**区域2：丛林/生长**
- 地图8×8
- 加入植物、蒸汽、燃烧状态
- 目标：500连
- 教学：学习状态交互与连锁触发

**区域3：高山/精炼**
- 地图10×10
- 加入矿石、冰晶、熔岩、带电状态
- 目标：2000连
- 教学：学习跨区域连锁与规则柱搭配

**区域4：火山口/终局**
- 地图12×12
- 全元素解锁，加入"催化剂尘"和"祝福"等高级状态
- 目标：10000连
- 挑战：利用全部机制，编织终极连锁引擎

## 4. 核心设计原则（确保"好玩"）

### 4.1 "我算到了"的实现
- 规则效果确定，无随机性——玩家放置的每条规则都会精确执行
- 系统提供"预演"功能（消耗资源）：玩家可以预览下一回合可能的演化方向
- 规则柱的位置和生效范围清晰可见，支持精确规划

### 4.2 "动态适配"的实现
- 手牌随机抽取，但保留机制让玩家有选择空间
- 世界演化中随机生成"扰动事件"（如"一阵风将光尘吹向东侧3格"），打破稳定状态，迫使玩家重新适配
- 没有唯一最优解——多种规则组合都能达成目标

### 4.3 "连锁引爆"的实现
- 规则之间的交互表是核心设计（详见下文）
- 鼓励跨区域规则搭配（如用蒸汽触发远处的植物增殖，植物增殖产生光尘，光尘又触发矿石转化）
- 早期的简单规则可以为后期的复杂爆发埋下伏笔

### 4.4 "连绵雨"的实现
- 积分系统是游戏核心反馈，而非单纯的胜负判定
- 演化阶段不可跳过，强制玩家观看自己的规划成果（类似音游打完后看回放）
- 音效和视觉的渐进式反馈，让"连绵"成为一种感官享受

## 5. 核心规则交互表（示例）

这是策划阶段最需要仔细设计的内容，确保元素交互有足够深度：

| 触发元素 | 目标元素 | 条件 | 结果 |
|---------|---------|------|------|
| 熔岩 | 水体 | 相邻 | 水体消失，生成蒸汽，熔岩变为岩石 |
| 蒸汽 | 植物 | 覆盖 | 植物触发"加速生长"（立即增殖一次） |
| 蒸汽 | 冰晶 | 覆盖 | 冰晶融化为水体 |
| 蒸汽 | 空气 | 占据 | 2回合后凝结为水体 |
| 光尘 | 岩石 | 覆盖 | 岩石转化为矿石（概率50%） |
| 光尘 | 植物 | 覆盖 | 植物产生"祝福"状态（下一回合额外增殖） |
| 带电 | 水体 | 覆盖 | 对水中所有元素造成"激活"（立即触发一次规则） |
| 带电 | 矿石 | 覆盖 | 矿石分裂，向相邻格生成新矿石 |
| 燃烧 | 植物 | 覆盖 | 植物消失，生成燃烧状态并向相邻植物蔓延 |
| 燃烧 | 冰晶 | 相邻 | 冰晶融化为水体，燃烧熄灭 |
| 祝福 | 任意 | 覆盖 | 该格规则柱生命延长2回合，且下次触发效果翻倍 |
| 催化剂尘 | 任意规则柱 | 覆盖 | 该规则触发所需条件降低（高级规则可提前触发） |

## 6. 技术实现建议

### 6.1 开发环境建议
- 引擎：Unity 或 Godot（2D网格游戏，两者都很适合）
- 语言：C#（Unity）或 GDScript（Godot）
- 平台：优先PC（Steam），后续可移植移动端

### 6.2 核心系统架构
```
GameManager（游戏总控）
├── WorldGrid（网格世界管理）
│   ├── Cell（单个格子，存储元素+状态）
│   └── ElementInteraction（元素交互处理器）
├── RuleEngine（规则引擎）
│   ├── RuleCard（规则卡牌数据）
│   ├── RulePillar（场上规则柱）
│   └── ChainReaction（连锁反应检测与执行）
├── PlayerSystem（玩家系统）
│   ├── HandManager（手牌管理）
│   ├── EnergySystem（能量管理）
│   └── PreviewSystem（预演系统）
├── UIManager（界面管理）
│   ├── ChainCounter（连锁积分显示）
│   └── FeedbackSystem（视觉/音效反馈）
└── LevelManager（关卡管理）
    ├── LevelData（关卡配置）
    └── BiomeSettings（生物群落设定）
```

### 6.3 连锁反应算法（伪代码）
```python
def execute_turn(world_grid, active_rules):
    chain_count = 0
    changed_cells = world_grid.get_all_cells()
    
    while changed_cells is not empty and chain_count < MAX_CHAIN:
        new_changes = []
        for each rule in active_rules:
            for each cell in changed_cells:
                if rule.condition_met(cell, world_grid):
                    result = rule.apply(cell, world_grid)
                    new_changes.extend(result.affected_cells)
                    chain_count += 1
                    
                    # 雪球机制：生成催化剂尘
                    if chain_count % 10 == 0:
                        random_cell = world_grid.get_random_cell()
                        random_cell.add_state("催化剂尘")
        
        # 检测终止条件
        if chain_count > STABILITY_THRESHOLD and no new_changes for 30s:
            break
        
        changed_cells = new_changes
        update_visual_and_audio(chain_count)
    
    return chain_count
```

### 6.4 关键设计参数（需调整）
- 初始手牌数：5
- 每回合抽牌数：3
- 初始能量：3
- 规则柱生命周期：4回合
- 连锁稳定性判定时间：30秒
- 催化剂尘触发间隔：每10连
- 灭绝规则触发阈值：单一元素超过总格数50%

## 7. 最小可行产品（MVP）范围

**第一版建议包含**：
- 1个生物群落（海岸）
- 4种基础元素（水体、岩石、土壤、蒸汽）
- 6种基础规则卡（3种转化、2种增殖、1种灭绝）
- 连锁积分系统（基础版）
- 3个教学关卡 + 2个挑战关卡
- 基础视觉反馈（颜色变化、简单粒子）

**预计开发周期**：2-3个月（单人全职）

## 8. 游戏特色总结（用于宣传）

> 《催化剂》是一款让你体验"上帝编织者"快感的策略规划游戏。你不是战士，而是规则的制定者。在微缩的网格世界中，放置转化规则，种下连锁反应的种子，然后静静地看着你的规划如何演变成一场席卷全屏的连绵风暴。从个位数到成千上万的连锁积分，每一次"叮"都是对你智慧的赞颂。如果你享受"我算到了"的智力快感，痴迷于《杀戮尖塔》的构筑和《陷阵之志》的预判，那么《催化剂》就是为你而生的游戏。
